Smouk — from scratch to “it runs”

One flow: what you do on your PC vs in the browser (Supabase). Open this page anytime while you work.

Blue = on your computer (Cursor, File Explorer, Terminal) Purple = in Supabase (or OpenAI) in the browser
Step 0

Prerequisites PC

Install Node.js (LTS) from nodejs.org. Restart the terminal after install. You need this for npm.

↓
Step 1

Your project folder PC

Your code lives in a folder (e.g. DSAIL-project/web). That folder has package.json — that means “this is the app.”

↓
Step 2

Supabase project (the cloud database + login) Browser

Create a free project at supabase.com. This is not a folder on your PC — it’s your backend online.

↓
Step 3

Copy the two Supabase strings Browser

Supabase → Project Settings → API → copy Project URL and Legacy anon key (eyJ…).

↓
Step 4

Create .env.local PC

Inside web/, create a file named .env.local. Paste URL + anon key + OpenAI key (see below).

↓
Step 5

OpenAI API key Browser

platform.openai.com → API keys → create key → paste into .env.local as OPENAI_API_KEY.

↓
Step 6

Run the SQL (tables + sample products) Browser

Supabase → SQL Editor → paste contents of web/supabase/schema.sql from your PC → Run.

↓
Step 7

Turn on Email login Browser

Authentication → Providers → Email. For dev, you can disable “confirm email” so login is instant.

↓
Step 8

Install & run the app PC

In Terminal: cd to web, then npm install, then npm run dev. Open http://localhost:3000.

↓
Step 9

Use the product PC

Sign up → Style passport → Shop chat. The chat talks to OpenAI + searches your real products table in Supabase.

Detailed clicks (expand)

What is .env.local?
A small text file in web/.env.local that only your computer reads. It holds the Supabase URL, anon key, and OpenAI key so the app knows which backend to use. Never paste these keys in chat or public GitHub.
Example .env.local (fill in your values) 
NEXT_PUBLIC_SUPABASE_URL=https://xxxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJ...
OPENAI_API_KEY=sk-...
Why two “worlds” (PC vs Supabase)?
  • Your PC = the website code (Next.js) you run with npm run dev.
  • Supabase (browser) = where accounts and product rows live permanently.
  • The keys connect them — like an address + a badge to enter your vault.
If you started from a completely new empty folder
You’d run npx create-next-app@latest . with TypeScript, then add Supabase packages and pages — we already did that for you in web/. You only need to configure env + SQL + run.
